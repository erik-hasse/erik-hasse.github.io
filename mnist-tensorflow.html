MNIST with TensorFlow

Posted on Sat 24 March 2018 in posts

In my previous post I used a number of different classifiers from Scikit-Learn to attempt to classify images from the MNIST dataset. The best choice of those that I tried was a random forest, which offered 97.3% accuracy on the validation set. However, I did not use any sort of neural network. Because of the complexity of even simple image recognition tasks such as this one, neural networks work really well. This is because in some sense they learn the features that are most useful for classification. Intuitively, we can describe some of these features. For example straight lines appear in 1, 4, and 7, or a closed loop means that it's a 6, 8, 9, or 0. However, it would be very difficult to turn these into features that we could feed into one of the simple models we used before!

I will be using TensorFlow to implement neural networks with different parameters. TensorFlow provides a built in estimator, DNNClassifier, that implements a basic feedforward neural network with the topology and activation function that we define. In this post, the topology is the main thing I'll be adjusting, although I will experiment with activation functions as well. After this, I'll set up a convolutional neural network. This is a bit more work, although TensorFlow still makes it relatively simple.

Setting things up¶

First we need to import the packages we'll need, define the functions we want to use to analyze the predictions, and get the data. Because we've already explored the data and know what it looks like, we don't need to do that again here. Notice that TensorFlow estimators (both the premade ones and the one I'll define later) provide estimators, but in order to maintain consistency with the previous post I'm going to use the same one as before.

In order to actually use TensorFlow, we need to setup a few things here. First we need to define the features that we'll feed into the model. Previously we thought of each example as having 784 different features, one for each pixel. In TensorFlow, we can pass this entire vector as a single feature, and even better we can pass it as a 28x28 array which more accurately represents the data. Within the model this is still treated as 784 features, so really this approach to the data is just to help us, not the estimator. In particular, we don't have the relative positions of the features that we gain with a convolutional neural network.

In [2]:
feature_columns = [tf.feature_column.numeric_column('x',shape=[28,28])]

Next, we need input functions. In TensorFlow, this is a function that takes no arguments and returns a batch of the data. However, after trying the estimator with the defined input function I received a type error. For reasons that I'm not sure of, the labels provided through TensorFlow have type uint8, which is not a type that TensorFlow supports as a label. So I first need to cast the labels to int32 to make everything work.

It is at this point that I also decide how I want to train the models. In general, the longer a model is trained the better it will fit the training data. However this may or may not lead to improved performance on the validation data either because the model is overfitting the training data, or does not have enough parameters to accurately model the data (underfitting). To see when a model is converging, I'll set the input function to cycle through all training examples 10 times. After each round of training I'll record the accuracy and then plot it to determine convergence. A batch size of 250 is used because my personal machine can process something of that size easily.

In [3]:
train_input_fn = tf.estimator.inputs.numpy_input_fn(
    x={'x': mnist.train.images},
    y=mnist.train.labels.astype(np.int32),
    num_epochs=10,
    batch_size=250,
    shuffle=True
)

In addition to an input function for the training data, I'm also going to define input functions for the validation and test data. The basic structure is the same, but we don't want to shuffle these because we want to compare them to the known labels, and we only need to look at each example once to make predictions on it.

Finally, the estimator's predict function provides a lot of information, but to evaluate the accuracy of the model I really only need the label. So this function takes in the result of predict() and returns a list of the predicted labels.

In [5]:
def get_labels(preds):
    return list(map(lambda x:x['class_ids'][0],preds))

Training the first model¶

For the first neural network classifier, I'm going to have just one hidden layer. Much like logistic regression formed a baseline for other classifiers, this will give us a starting point for neural networks. One rule of thumb for choosing the size of the single hidden layer is to take the mean of the number of features and number of labels. With 784 features and 10 labels, that gives us 397 hidden units. As discussed above, I'll train the model repeatedly to check when it converges with a plot.

In [ ]:
classifier = tf.estimator.DNNClassifier(n_classes=10,

Two hidden layers¶

With just one hidden layer, I was able to use a rule of thumb to get an estimate on how many nodes to use. With two or more layers it's harder to choose. We do know that the optimal number is likely between the size of the input layer (784) and the size of the output layer (10). I'm going to just try a few to see what works the best. However, it's quite possible we'll see no real improvements over the single hidden layer model. For many applications a single hidden layer is enough. For the most part the only way to find out is by experimenting, so that's what I'll do.

Instead of coming up with a new variable name for each classifier, I'm going to put them all in a list of two layer classifiers. This keeps the code cleaner.

Convolutional Neural Network¶

The models we've used so far all throw away one crucial piece of information: the relative positions of each pixel. A convolutional neural network provides a way to maintain this positional data. This is a bit harder to setup in TensorFlow than deep neural network, but still far easier than it would be to do manually.

To build a convolutional neural network, we'll need a model function, which tells TensorFlow how to process the data. We'll build this using the layers provided by TensorFlow. Convolutional neural networks provide a huge number of parameters to tun by hand, but I'm going to take a guess at them and we can consider tuning them based on the accuracy of the first guesses.

Between each layer I put the size of a single example in a comment just to keep things clear.

Note: Much of this is taken from the TensorFlow Layers tutorial. However that tutorial's layers can be adapted to many other convolutional neural networks.

The network architecture I've chose starts with two convolutional layers with 32 filters each, each with relu as an activation function, followed by pooling to bring the dimension down to 14 by 14 (by 32 for the depth). Then I repeat this with two 64 filter layers with relu and another pooling layer to bring the dimension to 7 by 7 (by 64). Finally, I have a dense layer with 1024 units.

In [5]:
def model_fn(features,labels,mode):
    
    input_layer = tf.reshape(features['x'],[-1,28,28,1])

    # 28 x 28 x 1
    
    conv1 = tf.layers.conv2d(
        inputs=input_layer,
        filters=32, 
        kernel_size=5,
        padding='same', # We want to keep the size the same through each conv layer
        activation=tf.nn.relu
    )

    # 28 x 28 x 32
    
    conv2 = tf.layers.conv2d(
        inputs=conv1,
        filters=32, 
        kernel_size=5,
        padding='same',
        activation=tf.nn.relu
    )
    
    # 28 x 28 x 32
     
    pool1 = tf.layers.max_pooling2d(
        inputs=conv2,
        pool_size=2,
        strides=2
    )
    
    # 14 x 14 x 32
    
    conv3 = tf.layers.conv2d(
        inputs=pool1,
        filters=64, 
        kernel_size=5,
        padding='same',
        activation=tf.nn.relu
    )
    
    # 14 x 14 x 64
    
    conv4 = tf.layers.conv2d(
        inputs=conv3,
        filters=64, 
        kernel_size=5,
        padding='same',
        activation=tf.nn.relu
    )
    
    # 14 x 14 x 64
    
    pool2 = tf.layers.max_pooling2d(
        inputs=conv4,
        pool_size=2,
        strides=2
    )
    
    # 7 x 7 x 64
    
    pool2_flat = tf.reshape(pool2,[-1,7*7*64])
    
    dense = tf.layers.dense(
        inputs=pool2_flat,
        units=1024,
        activation=tf.nn.relu
    )
    
    #This helps overfitting
    dropout = tf.layers.dropout(
        inputs=dense,
        rate=0.4,
        training=(mode==tf.estimator.ModeKeys.TRAIN)
    )
    
    logits=tf.layers.dense(
        inputs=dropout,
        units=10
    )
    
    predictions = {
        'class_ids': tf.argmax(input=logits,axis=1),
        'probabilities': tf.nn.softmax(logits,name='softmax_tensor')
    }
    
    if mode == tf.estimator.ModeKeys.PREDICT:
        return tf.estimator.EstimatorSpec(mode=mode,predictions=predictions)
    
    loss = tf.losses.sparse_softmax_cross_entropy(labels=labels,logits=logits)
    
    if mode == tf.estimator.ModeKeys.TRAIN:
        optimizer = tf.train.AdamOptimizer()
        train_op = optimizer.minimize(
            loss=loss,
            global_step=tf.train.get_global_step()
        )
        return tf.estimator.EstimatorSpec(mode=mode,loss=loss,train_op=train_op)
    
    eval_metrics_ops = {
        'accuracy': tf.metrics.accuracy(
            labels=labels,
            predictions=predictions['class_ids']
        )
    }
    
    return tf.estimator.EstimatorSpec(
        mode=mode,
        loss=loss,
        eval_metrics_ops=eval_metrics_ops
    )

Because we didn't really do any tuning on the hyperparameters, it's not surprising that our model has the same accuracy on the test set as it does on the validation set. Overall, this model shows the power of the convolution layers of a neural network for image recognition. In a network without convolution, we lose spatial information about which pixels are nearby. However, these relationships are very important, so just discarding the information is a bad idea.
